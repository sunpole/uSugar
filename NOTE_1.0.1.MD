# Техническая документация: Telegram-бот для семейного мониторинга сахара (Libre2) — версия 1.0.0+

## 1. Общее описание

Бот предназначен для семейной группы (родители+ребёнок с диабетом 1‑го типа).  
**Основные возможности:**

- **Распознавание скриншотов** из приложения Libre2 (уровень глюкозы, mmol/L или mg/dL).  
- **Расчёт доз инсулина** на еду, на коррекцию и их комбинацию с учётом персональных коэффициентов.  
- **Приём и обработка списка углеводов** по ключевым фразам («какой укол», «сколько укола», «посчитай укол») – расчёт ХЕ и дозы.  
- **Ручной ввод** замеров сахара, углеводов, ХЕ, инъекций (короткий и длинный инсулин), заметок.  
- **Журнал замеров** – вывод за день/неделю/месяц, при необходимости ссылка на файл.  
- **Напоминания** (проверка сахара, укол длинного инсулина, контроль через 2 часа после еды).  
- Работает **в личных сообщениях и в групповом чате** (требуются права администратора).

**Целевые версии:**  
- **1.0.0** – полнофункциональный бот на домашнем ПК/локальном сервере.  
- **2.0.0** – деплой на выделенный хостинг 24/7, персистентность состояний, webhook.  
- **3.0.0** – база продуктов с БЖУ и предрассчитанными дозами (в перспективе).

## 2. Стек технологий (полный, версия 1.0.0)

| Компонент               | Технология / Библиотека                | Назначение                                                                 |
|-------------------------|----------------------------------------|----------------------------------------------------------------------------|
| Язык                    | Python 3.12+                           |                                                                           |
| Telegram Bot API        | aiogram 3.x                            | Работа с Telegram (сообщения, группы, FSM, middleware)                    |
| OCR (распознавание)     | PaddleOCR (основной) + EasyOCR (резерв)| Извлечение числа глюкозы из скриншота. PaddleOCR точнее, EasyOCR – запасной |
| Предобработка фото      | OpenCV (cv2)                           | Контраст, бинаризация, удаление шумов перед OCR                           |
| Математика формул       | Python (float/Decimal)                 | Расчёт доз инсулина по персональным коэффициентам                         |
| Управление диалогами    | aiogram FSM (States, MemoryStorage / SQLiteStorage) | Пошаговые опросы (/setup, опросы после скриншота)    |
| Планировщик задач       | APScheduler (с персистентностью в БД)  | Напоминания, повторные опросы, события по расписанию                      |
| База данных (локальная) | SQLite + aiosqlite                     | Настройки, журнал, состояния FSM, напоминания                            |
| База данных (продакшн)  | PostgreSQL + asyncpg                   | Для версии 2.0.0                                                          |
| Логирование             | loguru                                 | Отладка и мониторинг ошибок                                               |
| Конфигурация            | pydantic-settings / python-dotenv      | Переменные окружения (токен, начальные параметры)                         |
| Тестирование            | pytest + pytest-asyncio                | Модульные тесты распознавания, формул, преобразований                     |

**Почему PaddleOCR:**  
- Даёт точность 92–95% на коротком цифровом тексте, устойчив к мелкому шрифту и артефактам сжатия.  
- EasyOCR оставлен как fallback с точностью ~85–88%.

## 3. Архитектура бота (обновлённая)

```mermaid
flowchart TB
    User[Пользователь в группе/ЛС] -->|сообщение, фото| TG[Telegram]
    TG -->|polling| Dispatcher[aiogram Dispatcher]
    
    subgraph Handlers
        PhotoHandler[Хендлер фото]
        CmdHandler[Хендлер команд]
        FoodHandler[Хендлер фраз про еду]
        FSMHandler[Хендлеры состояний]
    end
    
    subgraph Services
        OCRService[OCR: OpenCV → PaddleOCR/EasyOCR]
        FormulaService[Расчёт доз:
            - Коэф. чувствительности (ISF)
            - Углеводный коэф. (I:C ratio)
            - Добавки на приёмы пищи
            - Целевой сахар]
        JournalService[Журнал: запись / выборка]
        ReminderService[APScheduler + БД]
        ProfileService[Управление настройками]
    end
    
    subgraph Database
        SQLite[(SQLite / PostgreSQL)]
    end
    
    Dispatcher --> Handlers
    Handlers --> Services
    Services --> Database
    ReminderService --> Dispatcher
```

**Взаимодействие модулей:**

1. **PhotoHandler** – получает фото, вызывает OCRService → число сахара → FormulaService может предложить коррекцию (если актуален запрос).  
2. **CmdHandler** – команды `/start`, `/setup`, `/sugar`, `/carbs`, `/he`, `/insulin`, `/log`, `/help`.  
3. **FoodHandler** – распознаёт фразы «какой укол», «посчитай укол» и извлекает числа углеводов → FormulaService вычисляет ХЕ и дозу, при наличии актуального сахара добавляет коррекцию.  
4. **FSMHandler** – управляет многошаговыми диалогами (настройка профиля, опросы после скриншота).  
5. **Services** – выполняют бизнес‑логику и обращаются к БД.  
6. **ReminderService** – на старте поднимает все активные напоминания из БД.

## 4. Сценарии пользователя (глазами пользователя)

### 4.1 Добавление бота в семейную группу
1. Администратор группы добавляет бота по `@DiabetFamilyBot` и делает его **администратором** (нужно право «читать сообщения»).  
2. Бот пишет приветствие и предлагает выполнить настройку `/setup` (в группе или в личных сообщениях).

### 4.2 Первая настройка – профиль диабетика (команда `/setup`)
`/setup` запускает пошаговый опрос (FSM). Рекомендуется выполнять в личном чате с ботом, чтобы не засорять группу.  
Параметры настройки (привязаны к группе, в которой будет работать бот, либо к пользователю – указывается в начале):

1. **Единицы измерения сахара** `mmol/L` или `mg/dL`.  
2. **Целевой уровень глюкозы** (например, `5.5` mmol/L) – для расчёта коррекции.  
3. **Углеводный коэффициент (1 ХЕ = X г углеводов)** – обычно 10, 11 или 12.  
4. **Фактор чувствительности к инсулину (ISF)** – на сколько ммоль/л (или mg/dL) снижает сахар 1 единица инсулина. Может задаваться один на весь день **или три значения**:  
   - Утро (например, `2.5`)  
   - День (`3.0`)  
   - Вечер (`2.8`)  
5. **Инсулин/ХЕ коэффициент (I:C ratio)** – сколько единиц короткого инсулина нужно на 1 ХЕ. Может быть одинаковым или разным для завтрака/обеда/ужина.  
6. **Фиксированные добавки на основные приёмы пищи** (опционально):  
   - Завтрак: `+2.0` ед.  
   - Обед: `+1.5` ед.  
   - Ужин: `+2.5` ед.  
7. **Длинный инсулин (базальный)**:  
   - Доза (ед.)  
   - Время введения (например, `22:00`)  
   - Частота: 1 раз в сутки или 2 раза (тогда два времени, например, `08:00` и `20:00`).  
8. **Напоминания**:  
   - Бот предлагает добавить стандартные напоминания (измерение сахара перед едой, через 2 часа после еды, укол длинного). Можно настроить произвольные текстовые напоминания.  

После заполнения бот выводит сводку настроек и сохраняет их в БД для текущего чата.

### 4.3 Отправка скриншота Libre2
1. Пользователь делает скриншот и отправляет его в группу (или боту в личку).  
2. Бот распознаёт значение сахара (например, `12.3 mmol/L`).  
3. **Если контекст не связан с приёмом пищи:**  
   - Бот вычисляет коррекционную дозу:  
     `(текущий_сахар - целевой) / ISF_сейчас`  
     и пишет:  
     *«🔍 Сахар 12.3 mmol/L. Коррекция: (12.3-5.5)/2.5 = 2.7 ед.»*  
   - (Может сразу спросить про укол, если включён режим опроса.)  
4. **Если в течение последних 15 минут пользователь запрашивал расчёт еды** (или бот сам предлагает после скриншота), то бот использует актуальное значение сахара для комбинированной дозы.

### 4.4 Расчёт дозы на углеводы по фразе
1. Пользователь пишет в чат:  
   *«Какой укол 45 23 67 12»*  
   или  
   *«посчитай укол завтрак 34 56»*.  
2. Бот распознаёт ключевые слова (`какой укол`, `сколько укола`, `посчитай укол`) и извлекает числа.  
3. Суммирует углеводы: `45+23+67+12 = 147 г`.  
4. Переводит в ХЕ: `147 / 10 = 14.7 ХЕ`.  
5. Рассчитывает дозу на еду: `14.7 ХЕ * 1.5 ед/ХЕ = 22.05 ед` (коэффициент берётся из настроек, возможно с учётом приёма пищи).  
6. Добавляет фиксированную прибавку (если в сообщении фигурирует «завтрак», «обед», «ужин»): например, «завтрак» → `+2.0 ед`.  
7. **Если есть актуальное значение сахара (последний замер ≤ 15 мин назад):**  
   - Вычисляет коррекцию: `(текущий_сахар - целевой) / ISF_текущего_периода`.  
   - Итоговая доза короткого = доза на еду + коррекция.  
8. Бот отвечает подробным расчётом и итоговой рекомендацией:  
   *«📊 Углеводы: 147 г → 14.7 ХЕ.  
   🍽 Доза на еду: 14.7 × 1.5 = 22.0 ед + завтрак +2.0 = 24.0 ед.  
   🩸 Сахар сейчас 12.3 (15 мин назад). Коррекция: (12.3-5.5)/2.5 = +2.7 ед.  
   💉 Общая рекомендуемая доза: 26.7 ед.»*  
9. При желании бот может спросить: *«Ввести эту дозу в журнал?»* (FSM).

### 4.5 Ручной ввод данных
Пользователь может вносить записи без распознавания:

| Команда                | Пример                        | Действие                                                     |
|------------------------|-------------------------------|--------------------------------------------------------------|
| `/sugar 8.2`           | Запись сахара                 | Сохраняет замер в журнал, может предложить коррекцию         |
| `/carbs 45`            | Углеводы (г)                  | Пересчитывает в ХЕ и сохраняет, также запускает расчёт дозы  |
| `/he 5.5`              | ХЕ напрямую                   | Сохраняет ХЕ и предлагает дозу                               |
| `/insulin short 4.5`   | Укол короткого инсулина       | Записывает факт укола                                        |
| `/insulin long 18`     | Укол длинного                 | Записывает, может обновить напоминание                       |
| `/note Текст заметки`  | Произвольная запись           | Добавляет запись с временем и текстом                        |

### 4.6 Журнал замеров (просмотр истории)
1. Пользователь отправляет `/log` – бот спрашивает период: день/неделя/месяц (или можно `/log week`).  
2. Бот собирает записи из БД за указанный период:  
   - Сахар, ХЕ/углеводы, уколы, заметки – в хронологическом порядке.  
3. Формирует сообщение с отформатированным списком. Если сообщение превышает лимит Telegram (4096 символов), бот генерирует **CSV-файл** и отправляет его как документ.  
4. В перспективе (v3.0.0) – выгрузка в Google Sheets по ссылке.

### 4.7 Напоминания
- **Стандартные:**  
  - Измерение сахара перед завтраком/обедом/ужином.  
  - Укол длинного инсулина (за 5 минут до настроенного времени).  
  - Контроль через 2 часа после еды (автоматически ставится, когда бот зафиксировал приём углеводов и укол).  
- **Пользовательские:** любой текст и время (добавляются через `/remind 21:00 Проверь сахар перед сном`).  
- При пропуске ответа на опрос (например, «Сделали укол?») бот напомнит до 3 раз с интервалом 5 минут, после чего опрос снимается.

### 4.8 Обработка ошибок
- Если OCR не распознал число: *«Не удалось прочитать сахар. Попробуйте более чёткое фото или введите вручную /sugar»*.  
- Если в сообщении с просьбой расчёта нет чисел: *«Не найдены углеводы. Напишите числа через пробел после фразы»*.  
- Если профиль не настроен: *«Сначала выполните /setup»*.

## 5. Детали реализации основных функций

### 5.1 OCR и предобработка
- Используется PaddleOCR с языковой моделью `en` (цифры универсальны).  
- Предобработка: перевод в оттенки серого, адаптивная бинаризация, обрезка по координатам (можно задать маску области числа для Libre2).  
- Резервный EasyOCR активируется, если PaddleOCR не вернул число или возникла ошибка.

### 5.2 Модуль формул (FormulaService)
**Ключевые величины (хранятся в БД для каждого chat_id):**

- `target_glucose` – целевой сахар (float, mmol/L или mg/dL).  
- `isf` – фактор чувствительности (float) – **может быть один или разбит по времени**: `isf_morning`, `isf_day`, `isf_evening`. Время перехода задаётся (например, morning 6:00–12:00, day 12:00–18:00, evening 18:00–6:00).  
- `carb_ratio` – инсулин/ХЕ коэффициент (float). Может быть общий или раздельный для завтрака/обеда/ужина.  
- `add_breakfast`, `add_lunch`, `add_dinner` – фиксированные добавки (float).  
- `bread_unit_grams` – количество углеводов в 1 ХЕ (по умолчанию 10).  
- `units` – `mmol` или `mgdl` (для конвертации значений в формуле коррекции).  

**Алгоритм расчёта дозы на еду:**

1. Суммировать углеводы `C`.  
2. `HE = C / bread_unit_grams`.  
3. `meal_dose = HE * carb_ratio` (если ratio зависит от приёма пищи – выбрать соответствующий).  
4. Добавить `add_meal` для указанного приёма (если есть).  
5. Если передан текущий сахар `glucose` и он был измерен в течение последних 15 минут:  
   - `correction = (glucose - target_glucose) / isf_current` (isf_current выбирается по времени суток).  
   - `total = meal_dose + correction`.  
6. Вернуть `total`, а также промежуточные значения для объяснения.

**Для ручного ввода только сахара:**  
- `correction = (glucose - target_glucose) / isf_current`, рекомендация – эта доза.

### 5.3 Обработка фраз про «какой укол»
Бот анализирует входящее сообщение, если оно не команда и не фото.  
1. Ищет совпадение с регулярным выражением: `(какой укол|сколько укола|посчитай укол)`.  
2. Извлекает все числа из сообщения (через `re.findall(r'\d+\.?\d*')`).  
3. Определяет приём пищи по ключевым словам: `завтрак`, `обед`, `ужин`, `перекус`.  
4. Вызывает FormulaService, передавая список углеводов, тип приёма и chat_id (для извлечения настроек).  
5. Если настроек ещё нет – просит выполнить `/setup`.  
6. Если есть сохранённый «последний сахар» в пределах 15 минут – использует его для комбинированного расчёта, иначе вычисляет только еду.

### 5.4 FSM и опросы
**Состояния (в папке `states/`):**
- `ProfileSetup` – шаги настройки профиля.  
- `InjectionConfirm` – после рекомендации дозы, ожидание ответа (да/нет/доза).  
- `ReminderSetup` – добавление кастомных напоминаний.  
- `LogPeriod` – выбор периода для выгрузки журнала.

Хранение FSM в версии 1.0.0 – `MemoryStorage` (допускает потерю при перезапуске). Для продакшн – `RedisStorage` или `SQLiteStorage` с персистентностью.

### 5.5 База данных (SQLite)
**Основные таблицы:**

- `chats` – настройки на чат (chat_id, target_glucose, isf_morning, isf_day, isf_evening, carb_ratio, ... add_meal, bread_unit_grams, units, basal_dose, basal_time1, basal_time2).  
- `glucose_log` – (id, chat_id, user_id, timestamp, value, source: ocr/manual).  
- `food_log` – (id, chat_id, user_id, timestamp, carbs_grams, he, meal_type).  
- `insulin_log` – (id, chat_id, user_id, timestamp, insulin_type (short/long), dose).  
- `notes` – (id, chat_id, user_id, timestamp, text).  
- `reminders` – (id, chat_id, next_time, interval_minutes, message, active).  
- `pending_polls` – для FSM опросов (можно использовать встроенный FSM storage).

### 5.6 Планировщик (ReminderService)
- При старте бота загружает все активные напоминания из таблицы `reminders`.  
- Использует `APScheduler` с SQLiteJobStore для сохранения задач между перезапусками.  
- При наступлении времени отправляет сообщение в нужный чат.  
- Напоминание «через 2 часа после еды» создаётся динамически при записи укола короткого + приёма пищи и удаляется после выполнения.

## 6. Процесс разработки и отладки (на домашнем ПК, версия 1.0.0)

### 6.1 Установка окружения
```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install aiogram[redis,sqlite] opencv-python paddlepaddle paddleocr easyocr aiosqlite apscheduler loguru python-dotenv
```

### 6.2 Структура проекта (расширенная)
```
diabet_bot/
├── bot.py                   # точка входа, запуск polling
├── config.py                # .env (TOKEN, и т.п.)
├── handlers/
│   ├── photo.py             # хендлер фото
│   ├── commands.py          # /start, /help, /setup, /log, /sugar, /carbs...
│   ├── food.py              # фразы "какой укол"
│   └── fsm.py               # опросы (настройка, подтверждение)
├── services/
│   ├── ocr_service.py       # PaddleOCR + EasyOCR + OpenCV
│   ├── formula_service.py   # все расчёты
```

### 6.3 Чек-лист перед переходом на хостинг
- [ ] OCR распознаёт ≥ 10 разных скриншотов с разными условиями.  
- [ ] Все расчёты проверены вручную (еда, коррекция, комбинация).  
- [ ] Настройки сохраняются и корректно применяются.  
- [ ] Напоминания срабатывают в заданное время.  
- [ ] Опросы не зависают, корректно удаляются по таймауту.  
- [ ] Журнал выводится за разные периоды, работает экспорт в CSV.  
- [ ] Бот в группе реагирует только на целевые сообщения/фото (чтобы не мешать обычному общению).

## 7. Версия 2.0.0 – хостинг и эксплуатация (кратко)

- **Выделенный сервер** (TimeWeb, DigitalOcean, Hetzner) с Docker.  
- **Персистентность FSM:** Redis.  
- **Персистентность напоминаний:** APScheduler + SQLite/PostgreSQL JobStore.  
- **Webhook** через Nginx + Let’s Encrypt.  
- Мониторинг ошибок через sentry-sdk + лог-файлы.  
- Резервное копирование PostgreSQL.

## 9. Важнейшие правила и антипаттерны

| Чего избегать                                     | Почему                                                        | Правильный подход                                 |
|---------------------------------------------------|---------------------------------------------------------------|---------------------------------------------------|
| Хранение состояний в глобальных переменных        | Потеря при перезапуске                                        | FSM Storage (Memory/Redis)                        |
| Синхронное выполнение OCR в основном потоке       | Блокирует обработку других сообщений                          | `asyncio.to_thread` для OCR                       |
| Бесконечное ожидание ответа в FSM                 | Утечка памяти и зависшие опросы                               | Таймаут 5–10 мин, снятие опроса                   |
| Тихое игнорирование ошибок OCR/расчётов           | Пользователь не понимает, что произошло                       | Явные сообщения: «Не распознано. Введите вручную» |
| Жёстко зашитые формулы/коэффициенты               | Невозможно адаптировать без перезапуска                       | Всё в БД, чтение при каждом расчёте               |
| Реакция на любое фото или сообщение в группе      | Ложные срабатывания, раздражение пользователей                | Фильтр по ключевым словам/командам, проверка на скриншот |
| Игнорирование временной актуальности сахара       | Опасные рекомендации по старым данным                         | Использовать сахар только ≤15 мин давности        |
| Обработка нескольких одновременных опросов вразнобой | Путаница с контекстом                                       | Привязка FSM к `(chat_id, user_id)`               |

## 10. Заключение

Документация описывает Telegram-бота для помощи в ежедневном управлении диабетом 1‑го типа с акцентом на **семейное использование**. Версия 1.0.0 уже включает:
- полный расчёт доз инсулина (на еду, коррекцию, комбинацию);
- гибкую настройку персонального профиля;
- распознавание скриншотов Libre2;
- обработку сообщений с перечнем углеводов;
- журнал замеров и ручной ввод;
- напоминания и интерактивные опросы.

Бот развивается по модульному принципу, что позволяет заменять компоненты (OCR, БД, хранилище состояний) без переписывания всего проекта.
